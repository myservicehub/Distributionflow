# INVOICE GENERATION SYSTEM

## Overview
Automatic PDF invoice generation and email delivery system. Invoices are generated when managers confirm orders and are automatically emailed to retailers.

---

## 🎯 **Features**

### **Automatic Generation** ✅
- ✅ Invoice created when manager confirms/approves order
- ✅ Professional PDF format with branded template
- ✅ Unique invoice numbers (format: `INV-YYYYMM-XXXXXX`)
- ✅ Automatic calculation of totals, tax, discount
- ✅ Payment terms based on payment method

### **Email Delivery** ✅
- ✅ Automatically emailed to retailer (if email available)
- ✅ PDF attachment with professional email template
- ✅ From: `invoices@distributionflow.com`
- ✅ Branded email design matching your identity

### **Manual Actions** ✅
- ✅ View invoice in browser
- ✅ Download invoice PDF
- ✅ Resend invoice via email
- ✅ Access from orders page

---

## 🔄 **How It Works**

### **Automatic Flow:**

```
1. Sales Rep creates order
   ↓
2. Manager reviews order
   ↓
3. Manager clicks "Approve/Confirm" ✅
   ↓
4. System generates invoice PDF
   ↓
5. System emails invoice to retailer
   ↓
6. Invoice stored in database
   ↓
7. Retailer receives email with PDF attachment
```

### **Trigger Conditions:**
Invoice is generated when:
- Manager/Admin approves order (action: 'approve')
- Order status changes to 'confirmed' or 'approved'

**Code Location:** `/app/app/api/[[...path]]/route.js` (lines 1624-1643)

## 🔢 **Invoice Numbering**

### **Format:** `INV-YYYYMM-XXXXXX`

Examples:
- `INV-202406-A7B3C9` (June 2024, order prefix A7B3C9)
- `INV-202407-F2D8E1` (July 2024, order prefix F2D8E1)

**Generation Logic:**
```javascript
function generateInvoiceNumber(orderId) {
  const date = new Date()
  const year = date.getFullYear()
  const month = String(date.getMonth() + 1).padStart(2, '0')
  const orderPrefix = orderId.substring(0, 6).toUpperCase()
  return `INV-${year}${month}-${orderPrefix}`
}
```

## 🗄️ **Database Schema**

### **Invoices Table** (Optional - Create if needed)

```sql
CREATE TABLE invoices (
  id UUID PRIMARY KEY DEFAULT uuid_generate_v4(),
  order_id UUID REFERENCES orders(id) NOT NULL,
  invoice_number TEXT UNIQUE NOT NULL,
  invoice_date TIMESTAMPTZ DEFAULT NOW(),
  due_date DATE,
  total_amount DECIMAL(10,2),
  status TEXT DEFAULT 'sent', -- 'sent', 'paid', 'overdue'
  sent_at TIMESTAMPTZ,
  paid_at TIMESTAMPTZ,
  created_at TIMESTAMPTZ DEFAULT NOW()
);

CREATE INDEX idx_invoices_order_id ON invoices(order_id);
CREATE INDEX idx_invoices_number ON invoices(invoice_number);
```

## 🔧 **Troubleshooting**

### **Invoice Not Generated?**

**1. Check Order Status:**
```sql
SELECT id, order_status, delivery_status 
FROM orders 
WHERE id = 'order-uuid';
```

**2. Check Server Logs:**
```bash
tail -f /var/log/supervisor/nextjs.out.log | grep -i invoice
```

**3. Verify Order Approval:**
- Ensure manager/admin role approved order
- Check action was 'approve' or status changed to 'confirmed'

### **Email Not Sent?**

**1. Check Retailer Email:**
```sql
SELECT shop_name, owner_email 
FROM retailers 
WHERE id = 'retailer-uuid';
```

- If `owner_email` is NULL, invoice won't be emailed
- Update: `UPDATE retailers SET owner_email = 'email@example.com' WHERE id = 'xxx'`

**2. Check Resend API Key:**
```bash
echo $RESEND_API_KEY  # Should not be empty
```

**3. Check Resend Dashboard:**
- Go to https://resend.com/dashboard
- Check "Emails" tab for delivery status

### **PDF Generation Fails?**

**1. Check Dependencies:**
```bash
cd /app && yarn list @react-pdf/renderer
# Should show version 4.5.1 or higher
```

**2. Memory Issues:**
```bash
# Increase Node memory if needed
NODE_OPTIONS='--max-old-space-size=1024' yarn dev
```
